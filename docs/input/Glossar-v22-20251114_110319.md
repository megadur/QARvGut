1. # **A**

| Begriff | Bedeutung |
| :---- | :---- |
| Abtrag | hausinterner Postverteiler |
| AFSTD | Gremium Anforderungsmanagement und Standardisierung |
| ARSTD | Architekturmanagement und Standardisierung |
| Attribuierung | Benennung eines Dokuments während des Scannens |
| Anlagen  | sind nicht druckbare Unterlagen  |
|  Auftragsdaten |  Betrifft alle in rvSMD vorhandenen Daten, die für die Beauftragung relevant sind. z.B: Name des Gutachters, VSNR/Geburtsdatum Proband, rvPuR-Auftrags-ID etc. |
| Auftragsdokumente | Betrifft alle Dokumente, die bei Beauftragung an den Gutachter gehen sollen. Dazu zählen zum einen Dokumente die bei der Erstellung des rvPuR-Auftrages bereits in rvPuR vorhanden sind und zum anderen Dokumente, deren Erstellung im späteren Prozess aus rvSMD heraus getriggert werden |
| Authentifizierung | [Authentifizierung](https://auth0.com/de/intro-to-iam/what-is-authentication) bezeichnet das Verfahren, mit dem die Echtheit einer Tatsache oder eines Dokuments nachgewiesen wird. Im IT-Bereich wird dieser Begriff normalerweise mit dem **Nachweis der Identität eines Nutzers** verbunden. |
| Autorisierung | In der IT ist [Autorisierung](https://auth0.com/de/intro-to-iam/what-is-authorization) ein Prozess, der bestimmt, welche Aktionen ein authentifizierter Benutzer, ein System oder eine Anwendung innerhalb eines Systems oder Netzwerks ausführen darf.  Es geht also darum, die Zugriffsrechte festzulegen, nachdem die Identität (Authentifizierung) bereits festgestellt wurde.  |
